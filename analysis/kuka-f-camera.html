KUKA iiwa f-Camera Calibration Analysis

Dataset: RH20T cfg7 (KUKA iiwa)  |  Date: 2026-03-25

Known Issue: Cameras with serial numbers starting with "f" (e.g. f0271510) show a systematic skeleton offset from the actual robot. This is caused by poor base alignment calibration for these cameras. All f-prefix cameras are excluded from the rendering pipeline.

1. Summary

RH20T KUKA cfg7 has 8+ external cameras per scene. Of these, cameras with serial numbers starting with "f" (different camera model) consistently show a skeleton-to-image misalignment when rendered using base-frame FK. Digit-serial cameras (e.g. 104122060811) render correctly.

The root cause is the same as the UR5 calibration issue: the extrinsics_base_aligned transformation for f-cameras does not accurately map URDF base-frame positions to camera coordinates.

5. Root Cause

Same mechanism as the UR5 calibration issue:

RH20T extrinsics_base_aligned pipeline:
  1. Calibrate camera extrinsics relative to a charuco board
  2. Measure charuco board position relative to robot base
  3. Compose: camera_to_world = camera_to_charuco @ charuco_to_base

The f-prefix cameras are a different camera model. Their charuco_to_base measurement appears to have a systematic error, causing all f-cameras in a given calibration session to share the same offset.

ProblemRoot CauseScope
Skeleton offset on f-cameras Poor base alignment calibration for f-prefix camera model KUKA cfg7: all cameras with serial starting with "f"

6. Resolution

Decision: All f-prefix cameras are excluded from the KUKA cfg7 rendering pipeline. This is implemented as a filter in dataset/RH20T/adapter.py. Combined with the visibility threshold filter (>50%), this leaves camera 104122060811 as the primary rendering camera for KUKA cfg7.

EE-relative FK was evaluated as an alternative but not adopted because:

  • It introduces a ~25cm tool offset (TCP ≠ URDF flange)
  • The skeleton shape is correct but anchored at the wrong position
  • Filtering the bad cameras is simpler and gives better results on good cameras
