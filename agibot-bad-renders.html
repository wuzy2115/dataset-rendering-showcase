Root Cause Analysis

1. Skeleton Out of Frame (zero coverage)

Symptom: Gripper_scenario is completely black.

Cause: Head camera looks up/forward, both arms project below the image frame (y > 480). The old FOV check used axis-tip projections that could pass even when the full FK skeleton chain was out of frame.

Fix: Post-render pixel coverage check replaces axis-tip projection.

2. Skeleton Offset from Actual Arm Position

Symptom: Skeleton has correct shape but is shifted from the actual arm in RGB.

Cause: End-effector positions (state/end/position) are in robot base frame, but camera extrinsics are in world frame. When the robot base moves, the coordinate frames diverge. Example: Ep 698/900765 has robot at (0.22m, 0, 0) with yaw=0.08 rad, but calibration assumes a different base pose.

Fix needed: Transform camera extrinsics into robot base frame using per-frame state/robot/position and state/robot/orientation.

3. Wrong Camera Calibration Data

Symptom: Projection direction is completely wrong.

Cause: Some episodes have extrinsics/intrinsics from a different physical camera (external/overhead), not the head camera. Intrinsics (ppx=318, ppy=252) differ from task typical values (ppx=322-324, ppy=239-244), and camera translation ~2.5m from origin (too far for head-mounted).

Fix needed: Detect and filter episodes with mismatched camera calibration.
