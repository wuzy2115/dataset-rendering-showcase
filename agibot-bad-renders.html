AGIBot Rendering Issues Investigation

Coordinate frame mismatch between end-effector poses and camera extrinsics — full scan of 44,358 rendered episodes

Problem Overview

Core issue: End-effector positions (state/end/position) are in robot base frame, but camera extrinsics (head_extrinsic_params_aligned.json) are in world frame. The rendering code projects base-frame 3D points using world-frame camera parameters without any coordinate transform, causing misaligned skeleton overlays.

This manifests as two distinct failure modes depending on the relationship between the robot's odometry origin and the camera calibration origin.

Type A: Calibration Coordinate System Mismatch

Symptom: Skeleton rendering is completely wrong — projected bones appear in entirely wrong positions on the image, or the end-effector floats far off screen.

Cause: The camera extrinsics world origin does not coincide with the robot's odometry origin. The head camera (physically mounted on the robot at ~0.9m height) reports a world-frame position >1.5m from the robot's reported position — physically impossible for a head-mounted camera. This means the calibration board that defined the "world frame" was placed at a different location than where the robot's SLAM/odometry initializes its origin.

Diagnosis metric: ||cam_translation - robot_position|| > 1.5m

Impact: 2,102 episodes across 79 tasks. Not fixable by code — requires either re-calibration or finding the calibration-to-odometry transform for each affected scene.

Type B: Robot Base Offset (Fixable)

Symptom: Skeleton shape is roughly correct but shifted — bones track the arms but with a consistent positional offset.

Cause: The robot base has moved from the world origin (state/robot/position > 0.1m), but end-effector positions are still in robot base frame. The camera extrinsics are in world frame and close to the robot (cam↔robot ≤ 1.5m), confirming they share the same coordinate system. The missing step is transforming end-effector poses from base frame to world frame.

Fix: Before projection, apply the robot base transform: p_world = R_base @ p_base + t_base using per-frame state/robot/position and state/robot/orientation.

Impact: 6,653 episodes across 107 tasks.

Detection Methodology

Scan approach: For each of 44,358 rendered episodes (from dedup manifests), read state/robot/position[0] from proprio_stats.h5 and the first translation_vector from head_extrinsic_params_aligned.json. Compute:

• robot_offset = ||robot_position|| — how far the robot base is from world origin

• cam_robot_dist = ||cam_translation - robot_position|| — distance from camera to robot (should be ~0.9m for head-mounted camera)

Classification: Type A if cam_robot_dist > 1.5m. Type B if robot_offset > 0.1m and cam_robot_dist ≤ 1.5m.
