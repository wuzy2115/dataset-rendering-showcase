AGIBot Render Fix: Before / After

Robot base pose compensation fix for gripper_scenario projection

Root Cause

Bug: End-effector positions (state/end/position) are in robot base frame, but camera extrinsics are in world frame. When the robot base has non-zero position or orientation, the skeleton projection is offset or distorted.

Fix: Transform camera extrinsics into robot base frame using per-frame state/robot/position and state/robot/orientation before projection.
